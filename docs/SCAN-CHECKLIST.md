# Barcode scan verification checklist

Every one of the 106 symbologies is generated by a **reference encoder** (bwip-js / BWIPP, or JsBarcode), so each is standards-compliant *by construction*. Verification has two layers:

1. **Automated round-trip** — `npm run verify:decode` encodes → renders → decodes with ZXing → asserts decoded == input, for the 19 base symbologies ZXing can read. Marked **[auto]** below; no manual scan needed.
2. **Manual scan** (this checklist) — free decoders can't read the exotic long tail (4-state postals, MaxiCode, DotCode, Han Xin, Codablock-F, Mailmark, MSI, GS1 stacked/composite) or content variants (ZATCA/EPC/MeCard QR, GS1 Digital Link, HIBC, NTIN/PPN). Generate a sample on the site, scan it with a phone/handheld scanner, and confirm the decoded value matches.
