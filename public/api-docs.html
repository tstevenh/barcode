REST API Reference

Barcode APIs API

A public REST API that turns a URL into a barcode image. Send a GET request with your data and it returns a ready-to-use PNG or SVG — no SDK and no sign-up. Powered by bwip-js, supporting roughly 100 symbologies.

Overview

The Barcode APIs API turns a URL into a barcode image. Send a GET request with your value and options as query parameters, and the API responds with the image. There is nothing to install and nothing to host.

  • Returns real image files — PNG (raster) or SVG (vector).
  • Stateless: every request is independent, driven entirely by query parameters.
  • Public and open — no API key or sign-up required.
  • CORS enabled, so you can call it directly from browser front-ends.

Authentication

The API is public — no authentication, API key, or sign-up is required. Just send a request to any endpoint over HTTPS. Please use it fairly; see Rate limits.

Base URL

All requests are made over HTTPS against the base URL below. Append each endpoint path to it.

Base URL
https://barcodeapis.com

  2. Build the URLAppend your value and any options as query parameters.
  3. Use the imageOpen it, embed it in an <img> tag, or download it.

HTML

Endpoints

GET/barcode

Returns a barcode image. Defaults to PNG; set format=svg for vector output.

GET/barcode.png

Always returns a PNG image, regardless of the format parameter.

GET/barcode.svg

Always returns a scalable SVG image. Ideal for print and high-resolution use.

GET/health

Health check. Returns a small JSON payload to confirm the API is up.

Query parameters

All parameters are passed in the query string. Only data is required.

ParameterDefaultDescription
data required—The value to encode.
typeqrcodeSymbology id (see Symbology types).
formatpngOutput format: png or svg.
scale3Pixel density / module size (1-20).
heightautoBar height in millimetres (linear codes).
includetextoffSet to 1 to print the human-readable text.
textaligncenterText alignment: left, center or right.
color000000Foreground hex colour (no #).
bgffffffBackground hex colour (no #).
padding0Quiet-zone padding around the symbol.
rotateNRotation: N, R, L or I.
columnsautoData column count (PDF417).

Responses & errors

On success the API responds with the image and the matching content type (image/png or image/svg+xml). On failure it returns a plain-text message with an appropriate HTTP status.

StatusMeaning
200Success — image returned.
400Missing data, unknown type, or invalid value for the symbology.
429Too many requests — you hit the rate limit; retry shortly.
404Unknown route.
500Unexpected server error.

Rate limits

The public API is rate limited so it stays fast for everyone. If you exceed the limit you receive 429 Too Many Requests — wait a moment and retry.

  • Cache generated images instead of requesting the same barcode repeatedly.
  • Prefer SVG when one image must scale to many sizes.
  • Batch requests and avoid tight retry loops.
💡 Generated barcodes are static images, so caching them is the easiest way to stay within the limits.

CORS

Every response includes Access-Control-Allow-Origin: *, so browser-based apps on any origin can load images directly from the API — no proxy or extra configuration required.
